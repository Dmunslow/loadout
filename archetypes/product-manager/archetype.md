# Product Manager Archetype

Default rules and agents for product managers. Covers writing requirements,
turning ideas into stories, reviewing UX decisions, and handling day-to-day
communication.

## What's included

**Rules**
- `product-thinking` — always frame work in terms of user problems; never jump to solutions
- `documentation-style` — structured docs with problem, context, success criteria, out of scope
- `boundaries` — describe what's needed, don't write or edit code directly

**Agents**
- `prd-writer` — takes a rough idea and produces a structured product requirements document
- `story-writer` — converts requirements into user stories with acceptance criteria
- `ux-reviewer` — reviews designs and flows against product goals and user needs
- `personal-assistant` — handles Slack messages, emails, and calendar scheduling using connected tools
