## Plan before coding
- Always outline the approach and confirm it before writing implementation code
- For anything non-trivial, break the work into phases with clear checkpoints

## Pull requests
- One concern per PR — don't mix features, refactors, and bug fixes
- Write a clear description of what changed and why, not just what
- No TODO or FIXME comments in merged code — finish the thing or file a ticket

## Code review
- Treat review comments as questions, not criticisms
- Every change needs at least a self-review pass before asking for another person's time

## Dependencies
- Don't add a dependency for something that can be done simply in-house
- Check the licence before adding any third-party package
